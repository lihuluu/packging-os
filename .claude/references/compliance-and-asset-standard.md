# 合规与素材授权标准

## 核心原则

- 包装用于商业销售前，合规信息和素材授权必须被显式检查，不能只停留在设计稿里“看起来没问题”。
- 本标准不替代企业法务、品控、法规或供应商的最终审核；它用于让设计流程提前暴露缺口。
- 未经确认的法规文案、标签信息、字体、图片、插画、AI 生成图和第三方素材，不得标记为可进入最终交付。
- 如果资料来源、授权范围或审批人不清楚，必须标为 `待确认`，不能默认可用。

## 合规检查范围

茶叶包装项目至少检查：

| 类型 | 检查内容 | 设计侧动作 |
| --- | --- | --- |
| 产品基础信息 | 品名、茶类/工艺、规格/净含量、配料或原料说明 | 在包装信息区预留完整层级 |
| 生产与责任信息 | 生产商、委托方、产地、生产许可证/SC、执行标准 | 标注待法规/品控确认，不自行编写 |
| 流通信息 | 条码、保质期、贮存条件、生产日期位置 | 在版面中预留可变信息区 |
| 宣传边界 | 功效、健康、产地、年份、奖项、认证等表述 | 无证明材料时不得当作确定卖点 |
| 食品接触与内包材 | 铝箔袋、茶罐、内托、封口材料是否涉及食品接触 | 要求供应商提供材料说明或合规文件 |
| 渠道/地区 | 仅大陆销售、出口、跨境或特渠 | 未确认销售范围时不得给最终合规结论 |

## 素材授权检查范围

| 素材类型 | 必须记录 | 风险提示 |
| --- | --- | --- |
| 字体 | 字体名称、授权来源、商用范围、是否可嵌入/转曲 | 免费字体不等于可商用 |
| 图片/摄影 | 来源、摄影师/图库、授权范围、是否可用于包装销售 | 网络参考图不得直接用于成品 |
| 插画/图标 | 作者、授权协议、可修改范围、使用期限 | 二次修改也可能受授权限制 |
| AI 生成图 | 生成工具、提示词记录、后期处理、企业内部使用规则、项目内归档路径 | 不得冒充真实产地/真实工艺/真实检测或认证证据 |
| 品牌资产 | Logo、VI、认证标志、合作方标志 | 使用比例、留白和颜色需按权利方规范 |
| 竞品/参考素材 | 来源和用途 | 只能用于分析，不得进入成品文件 |

## 授权状态

| 状态 | 含义 | 是否可进入最终交付 |
| --- | --- | --- |
| `已确认` | 来源、授权范围和使用方式均已确认 | 可以 |
| `内部资产` | 企业自有或企业已购买资产，但需记录来源 | 可以，需归档 |
| `待确认` | 来源或授权范围不完整 | 不可以 |
| `仅参考` | 只用于 moodboard、竞品分析或方向参考 | 不可以 |
| `不可用` | 明确无商用授权或权利不明 | 不可以 |

## 输出要求

涉及视觉系统、插画、提案、印前或最终交付时，必须输出：

- `合规待确认项`
- `素材授权状态`
- `不能进入最终交付的素材`
- `需要谁确认`
- `默认归档位置`

推荐归档：

- 素材授权登记：`03_Design/02_Assets/asset-register.md`
- AI 概念素材草案：`03_Design/02_Assets/AI_Concepts/{task-id}/`
- 合规检查记录：`03_Design/06_Proof_Record/compliance-review.md` 或并入 `prepress-review.md`
