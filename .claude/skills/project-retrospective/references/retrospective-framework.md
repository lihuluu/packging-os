# 项目复盘框架

## 目录
- 输入来源
- 共享字段词表
- 根因追问
- 阶段映射
- 输出模板

## 输入来源

- 上市反馈
- 销售或客户反应
- 供应商或生产问题
- 内部复盘笔记
- 时间线或审批问题

## 共享字段词表

复盘与项目卡、知识沉淀统一使用以下公共字段：
- `项目摘要`
- `来源文档`
- `已知事实`
- `关键假设`
- `风险`
- `开放问题`
- `结论沉淀`
- `下一步动作`

如需完整定义，阅读 [../../../references/shared-field-glossary.md](../../../references/shared-field-glossary.md)。

## 根因追问

- 问题来自于 brief 不清、假设错误、概念选择不当、结构不匹配、材质不匹配，还是印前控制失误？
- 哪个团队最早知道这个风险？
- 本来应该在哪个检查点发现？
- 这是重复问题还是偶发问题？

## 阶段映射

把问题映射到：
- 洞察层
- 概念层
- 结构层
- 视觉层
- 材料层
- 印前层
- 协作层
