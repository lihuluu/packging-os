# Proposal Builder Framework

## 目录
- 提案类型

- 常见页结构
- 风险处理

## 提案类型

先判断当前提案属于哪一类：
- 方向选择型：还在比较 2 到 3 个方向，重点是差异和推荐逻辑
- 方案确认型：主方向已经选定，重点是说明方案如何落地以及需要拍板什么
- 纠偏型：项目卡住了，需要重新解释为什么原方案要调整

## 页序原则

- 先问题，再洞察，再方案
- 先推荐路径，再备选路径
- 每一页只承担一个决策任务
- 涉及成本、量产、法规的风险单独成页或单独成段

## 常见页结构

### 方向选择型
1. 项目背景
2. 关键洞察
3. 方向对比
4. 推荐方向
5. 结构与视觉影响
6. 风险与待确认项

### 方案确认型
1. 项目背景
2. 方案概述
3. 结构说明
4. 视觉规则
5. 材质工艺建议
6. 量产风险
7. 待拍板事项

### 纠偏型
1. 当前问题
2. 问题根因
3. 调整方案
4. 影响评估
5. 需要重新确认的事项

## 风险处理

- 用 `已确认` 和 `待验证` 分开写
- 成本、交期、供应商能力、公差和法规是高优先级风险
- 如果某页依赖其他团队输入，写清负责人和时点
