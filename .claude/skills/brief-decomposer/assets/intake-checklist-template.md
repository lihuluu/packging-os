# Brief 信息需求清单（通用茶叶）

> **用途**：在 Brief 阶段向项目各相关方（产品 / 研发、合规、市场、设计、采购）批量收集尚缺的关键信息。
> **项目**：{项目名称}
> **日期**：{YYYY-MM-DD}
> **当前进度**：{当前阶段，例如：前期沟通 Brief / 已进入结构层 / 发厂前复核}
> **品类备注**：{调味茶 / 紧压茶 / 袋泡茶 / 礼盒 / 单品 SKU 等 —— 影响下方"按品类选填"小节的取舍}

## 使用说明

1. 本表用于**主动收集**信息，不替代 `brief-decomposition.md`（过程分析）与 `packaging-brief.md`（任务书快照）；收回的信息必须回写到这两份文档，本表不作为最终结论。
2. 按当前项目品类勾选"附录：按品类选填"；不适用的小节整段删除。
3. P0 / P1 / P2 是收集顺序参考；如果某项 P1 反而是当前瓶颈，可上调。
4. 表格行数已编号，便于回填时按号对接。

## 优先级说明

- 🔴 **P0**：尽快提供（直接阻塞方向 / 结构 / 成本判断）
- 🟠 **P1**：设计审核之前必须明确（影响视觉、合规、生产）
- 🟢 **P2**：可后补，有则更好
