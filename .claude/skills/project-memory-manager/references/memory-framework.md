## 记录目标

项目卡的作用不是复述全部过程，而是让任何一个后续 skill 都能快速回答这四件事：
- 这个项目现在在什么阶段
- 哪些条件已经确认
- 哪些问题还没解决
- 下一步最该推进什么

## 开案必查项

新项目或阶段重启时，至少先确认这些关键项：

- 实物规格 / 内容物
- 目标上市时间
- MOQ 预估
- 环保优先级
- 当前供应商状态
- 合规/标签负责人
- 素材授权责任人
- 样品验收负责人

如果这些项中有任何一项缺失：

- 不要假装项目状态已经完整
- 必须写进 `开放问题` 或 `关键假设`
- 如果它会影响结构、材质、报价或打样，优先提高风险等级

## 共享字段词表

项目卡与复盘、知识沉淀统一使用以下公共字段：
- `项目摘要`
- `已知事实`
- `关键假设`
- `风险`
- `开放问题`
- `下一步动作`

如需完整定义，阅读 [../../../references/shared-field-glossary.md](../../../references/shared-field-glossary.md)。

## 更新规则

### 什么时候更新
- 新项目开案后
- 会议结束后
- 概念、结构、材质或供应商决策发生变化后
- 打样反馈或报价反馈回来后
- 上级或客户重新定义目标后

### 怎么更新
- 新确认的信息进入 `已知事实`
- 失效的信息进入 `本次更新 -> 失效`
- 仍未证实但足以影响判断的信息进入 `关键假设`
- 自相矛盾的信息进入 `开放问题` 并标明冲突来源

### 记录粒度
- 只记录会影响后续判断和推进的信息
- 不把聊天原文整段复制进项目卡
- 每次更新都要指出本次变化点

## 风险信号

发现以下情况时应在项目卡里单独标记：
- 目标价格带与结构/工艺野心冲突
- 上市时间紧，但仍有关键冻结项未定
- 渠道已定，但运输或陈列要求不明确
- 已经进入打样，但主方向仍在变动
- 供应商能力、MOQ 或交期尚未确认
- 法规、条码、强制信息仍未锁定
- 字体、图片、插画、AI 图或合作方标志授权状态不明
- 白样/彩样未验收，却准备进入发厂或量产
- 开案必查项缺 2 项及以上，但项目已进入结构、材质或供应商沟通
