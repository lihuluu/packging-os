# 项目推进框架

## 目录
- 常见里程碑
- 关键路径判断
- 节点风险
- 推进规则
- 输出模板

## 常见里程碑

包装项目通常至少会经过这些节点：
- brief 确认
- 研究/定位确认
- 概念方向确认
- 结构确认
- 视觉确认
- 材质工艺确认
- 供应商确认
- 打样
- 合规/标签复核
- 素材授权确认
- 白样验收
- 彩样/合同样验收
- 大货首件/抽检
- 发厂
- 上市或交付

## 关键路径判断

优先关注这些常见关键路径：
- brief 未定 -> 后续所有节点延迟
- 结构未定 -> 无法稳定推进视觉和材质
- 材质/工艺未定 -> 无法发出准确询价或印前文件
- 供应商未定 -> 打样和量产节奏不可控
- 法规文案未定 -> 发厂前高风险返工
- 素材授权未定 -> 最终交付和商业上市受阻
- 白样/彩样未验收 -> 大货风险不可控

## 节点风险

以下节点应默认设为高风险：
- 首次打样
- 多工艺叠加项目
- 多 SKU 联动换版
- 节假日前后的排产窗口
- 涉及外部审批或跨国供应链的项目
- 标签/法规、素材授权或食品接触材料仍待确认
- 白样、彩样或大货首件存在有条件通过

## 推进规则

- 每个里程碑都要有负责人、目标时间和前置条件
- 待审批状态不能记作完成
- 外部依赖必须单独列出，不要混在内部任务里
- 合规、授权、白样、彩样和大货验收节点必须有负责人和状态，不要写成普通提醒
- 当日期不确定时，先写顺序和依赖，不伪造精确日期
- 每次更新都要指出当前最大的单点瓶颈
