# Quotation Comparator

## 概述

把多家供应商报价从“价格列表”转成“可决策比较”。重点不是找最低价，而是识别真实总成本、执行风险和能力匹配度。

## 典型触发句

- 帮我比较这几家包装供应商的报价。
- 哪家工厂更适合这个项目？
- 这个报价单里有什么风险？
- 请整理一个供应商对比表。
- 这家便宜很多，真的更划算吗？
- 帮我看看这几份报价是不是一个口径。
- 这几家供应商该怎么选更稳？

## 使用流程

1. 提取各报价中的单价、MOQ、版费、刀模费、打样费、交期和付款条件。
2. 对齐同一口径，标出哪些报价范围、工艺或材质并不一致。
3. 比较价格之外的关键因素：工艺能力、稳定性、交付风险、沟通效率和隐含成本。
4. 形成推荐方案、备选方案和需要补问的问题。
5. 如无法直接决策，指出最小补充信息集合。

## 比较规则

- 先确认报价是不是同口径，再比较价格。
- 对明显缺项或描述模糊的报价，不要默认已经包含。
- 低价如果依赖降材、减工艺、缩检验或更长交期，要直接写成风险。
- 同时考虑打样阶段成本和量产阶段成本，不只看单盒价格。
- 如果供应商能力与项目复杂度不匹配，即使便宜也要降低推荐级别。
- 报价比较必须标注每份报价的成本证据等级 [C3-C5]；低于 C3 的不能作为价格比较依据
- 没有同口径（含税/含运/含打样/含刀模/含版费/含损耗/无外协）时，输出结论必须是："当前报价不可直接比较，需先补齐口径"

## 输出要求

- 输出比较表，至少包含 `价格`、`MOQ`、`交期`、`工艺能力`、`风险` 和 `推荐判断`。
- 补充 `需要补问的问题` 和 `建议下一步`。
- 如果没有一家明显合适，要明确说明而不是强行推荐。

## 输出后检查

- 如果本次比较确认了主供应商候选、改变了 MOQ、交期、预算、工艺能力判断或新增了风险项，必须提示更新项目卡。
- 提示格式使用：`⚠️ 本次任务改变了 [具体项]，建议更新项目卡：project-memory-manager`
