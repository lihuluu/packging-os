# 视觉系统框架

## 目录
- 核心输入
- 商业沟通目标
- 包装面任务与信息层级
- 阅读顺序与低保真结构
- 设计规则分类
- 效果图规划
- 输出模板

## 核心输入

- 已选概念方向
- 结构形式
- 必须出现的信息与卖点
- 品牌资产与既有规范
- 渠道背景
- 目标人群与价格带
- 法规/背标信息负担
- 字体、图片、插画、AI 生成图、认证标志和合作方 Logo 的来源与授权状态

## 商业沟通目标

视觉方向单先回答这些问题，再进入风格：

- 第一眼必须被识别的内容是什么？
- 第二眼必须让消费者相信什么？
- 哪些内容不能被误读或弱化？
- 购买场景是电商主图、搜索列表、货架、礼赠开箱还是复购识别？

## 包装面任务与信息层级

优先定义这些面的优先级：
- 正面
- 顶部或盒盖
- 侧面
- 背面信息区
- 内页或内托

每个重要包装面必须写清任务和信息层级：

| 面别 | 商业任务 | 一级信息 | 二级信息 | 三级信息 | 功能/合规信息 |
| --- | --- | --- | --- | --- | --- |
| 正面 | 吸引/识别/记忆 | | | | |
| 背面 | 解释/信任/合规 | | | | |
| 侧面/顶部 | 辅助识别/陈列展示 | | | | |

常用判断：
- 正面解决“我是谁、为什么值得看”。
- 背面解决“为什么可信、怎么使用、怎么买得放心”。
- 侧面/顶部是否重要，取决于货架陈列、电商角度和结构展示方式。

## 阅读顺序与低保真结构

为正面、背面分别写出阅读顺序，例如：

- 正面：核心识别 -> 品牌背书 -> 品类/规格 -> 辅助认证
- 背面：产品解释 -> 信任依据 -> 使用/成分/存储 -> 条码/厂家/法规

需要给后续设计师一个文字低保真结构，但不要在方向单阶段过早锁死精确尺寸：

```text
[一级信息]
[主视觉/核心图形]
[二级信息]
[三级/功能信息]
```

## 设计规则分类

- 字体层级
- 色彩系统
- 图像或插画角色
- 图形语言与边框逻辑
- 信息密度
- SKU 系统规则

## 效果图规划

明确：
- 需要哪些角度
- 哪些细节需要特写
- 是场景图还是纯净白底展示
- 哪些纹理、工艺或开盒动作必须被看见

效果图按验证目的写，不只按“好看角度”写：
- 电商主图：验证第一眼识别
- 搜索/货架远视：验证远距差异化
- 背面平铺：验证信息可读与法规区
- 细节特写：验证材质、印刷、工艺是否支撑概念
- 展开图或结构图：验证各面信息分配是否正确

## 素材授权检查

视觉方向进入最终交付前，必须检查素材授权状态：

| 素材类型 | 检查内容 | 状态 |
| --- | --- | --- |
| 字体 | 字体名称、来源、商用范围、是否可嵌入/转曲 | 已确认 / 内部资产 / 待确认 / 仅参考 / 不可用 |
| 图片/摄影 | 来源、授权范围、是否可用于包装销售 | 已确认 / 内部资产 / 待确认 / 仅参考 / 不可用 |
| 插画/图标 | 作者、协议、可修改范围、使用期限 | 已确认 / 内部资产 / 待确认 / 仅参考 / 不可用 |
| AI 生成图 | 工具、提示词记录、企业内部使用规则 | 已确认 / 内部资产 / 待确认 / 仅参考 / 不可用 |
| 认证/合作方标志 | 权利方规范、颜色、留白、使用范围 | 已确认 / 内部资产 / 待确认 / 仅参考 / 不可用 |

权利不明的素材只能用于参考或草案，不得进入 `04_Final`。

## 长度控制

- 主文档用于决策，优先控制在 1-2 页。
- 执行细节进入附录，避免客户或项目负责人被过多技术信息淹没。
- 如果用户只要求快速判断，输出主文档即可；如果用户要求交给设计师执行，再补附录。

## 边界

- 不替代 `material-finishing-advisor` 判断真实材质和工艺组合。
- 不替代 `supplier-brief-writer` 输出正式供应商 brief 或询价口径。
- 不替代 `prepress-checker` 输出最终印前合规结论。
- 成本只写压力来源和 [C0] 级推断，不输出真实价格。
