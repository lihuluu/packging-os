# Visual System Builder

## 概述

把概念意图转成具体可执行的包装视觉规则。重点是商业沟通目标、包装面任务、信息层级、AI 概念素材任务和量产可行的视觉方向，而不是空泛的风格形容词。

本 skill 服务 Claude Code 的 `.claude/skills` 工作流：它负责让视觉方向足够指导后续设计执行，但不替代材料、供应商、印前或提案 skill 的最终判断。

## 典型触发句

- 帮我做这个包装的视觉系统。
- 这个包装正面应该怎么排信息层级？
- 请把这个概念转成可执行的包装视觉方向。
- 我需要一份包装风格板/视觉规则说明。
- 这个系列包装的 SKU 视觉系统应该怎么统一？
- 这个包装正面和整体风格应该怎么落？
- 这个包装正面第一眼应该看到什么？
- 这个包装背面信息层级应该怎么处理？
- 帮我把这个方向落成版式和视觉规则。
- 这份视觉方向单不够商业/不够可执行，帮我优化。
- 视觉方向单里的图像和图形可以交给 AI 做吗？
- 把这个视觉方向拆成 AI 概念素材任务。
- 给我生成主视觉、辅助图形、底纹或印章的 AI 草案任务。

## 使用流程

1. 确认已选概念、结构形式、渠道场景和必须上包装的信息。
2. 先定义商业沟通目标：第一眼卖什么、第二眼让人相信什么、哪些内容不能误读。
3. 使用 [visual-framework.md](./references/visual-framework.md) 定义包装面任务、正背侧面信息层级、阅读顺序和低保真结构。
4. 把品牌意图转译成字体、色彩、图像、信息密度和 SKU 规则。
5. 如果图像/图形适合由 AI 生成草案，拆成 `AI 概念素材任务`，明确任务 ID、素材类型、生成目的、输入依据、输出数量、禁止内容、归档路径和默认资产状态。
6. 检查字体、图片、插画、AI 生成图、认证标志、合作方 Logo 等素材的来源和授权状态。
7. 输出主文档优先：控制在决策友好的短视觉方向单；执行细节、AI 任务和授权清单放入附录或 `asset-register.md`。
8. 如果某些视觉想法依赖材质或工艺，要明确写出来，但不要替材料层或供应商层下最终结论。
9. 如果视觉判断依赖货架识别、材质气质或印刷效果，按 [../packaging-os/references/bridge-node-rules.md](../packaging-os/references/bridge-node-rules.md) 提示是否应插入 `货架模拟`、`概念到材质翻译` 或 `供应商可行性确认`。

## 设计规则

- 先做商业沟通任务，再做信息层级，最后做风格装饰。
- 优先考虑真实购买场景下最重要的包装面。
- 正面通常负责吸引、识别和记忆；背面通常负责解释、信任和合规；侧面/顶部按结构和渠道承担辅助识别或展示任务。
- 信息密度要和目标人群行为及价格带匹配。
- 色彩和特殊效果要和真实印刷条件相匹配。
- 如果法规文案过多挤压了视觉表达，要直接指出并建议重新分配层级。
- 如果某个视觉方向只有在特定材质、表面反射或货架距离下才成立，必须显式写出桥接依赖。
- AI 可以生成主视觉草案、辅助图形、底纹、印章形态、图标、植物/产地意象和风格变体；生成结果默认只用于提案筛选、方向验证或设计师重绘参考。
- AI 不得生成或替代 Logo、认证标志、真实检测证明、真实产地照片、法规证据，不能冒充真实产地、真实工艺或真实背书。
- AI 概念素材默认归档到 `03_Design/02_Assets/AI_Concepts/{task-id}/`，不得直接归入 `04_Final`。
- 不要在视觉方向单里输出真实成本、供应商一定能做、或印前最终合规这类越权结论；只能标注风险、依赖和成本压力 [C0]。
- 权利不明的字体、图片、插画、AI 图、认证标志或合作方 Logo 只能标为 `待确认`、`仅参考` 或 `不可用`，不能进入最终交付。
- AI 生成草案在企业内部规则和生成工具授权未确认前，授权状态不得写为 `已确认`；默认写为 `AI 生成草案 / 仅参考 / 不可直接终稿`。
- 视觉方向如使用素材，应按 [../../references/compliance-and-asset-standard.md](../../references/compliance-and-asset-standard.md) 输出授权状态；需要成表时使用 `asset-register.md`。

## 输出要求

- 输出 `商业沟通目标`、`包装面任务分配`、`信息层级`、`阅读顺序`、`视觉规则` 和 `效果图/场景验证`。
- 当用户要求 AI 处理图像/图形，或视觉方向明显需要图像/图形探索时，输出 `AI 概念素材任务`。
- 主文档优先短而可读；字体、尺寸、线宽、留白、印刷风险等细节放入附录。
- 说明哪些元素在 SKU 间应固定，哪些可以变化。
- 输出 `素材授权状态`：已确认 / 内部资产 / 待确认 / 仅参考 / 不可用。
- AI 概念素材任务必须包含：任务 ID、素材类型、生成目的、输入依据、输出数量、禁止内容、归档路径、默认资产状态。
- 如果材质或工艺还没定，最后补充开放依赖项。

## 输出后检查

- 如果本次任务冻结了关键视觉规则、包装面优先级、法规信息层级、素材授权状态、依赖的材质/工艺前提，或暴露了新的量产风险，必须提示更新项目卡。
- 提示格式使用：`⚠️ 本次任务改变了 [具体项]，建议更新项目卡：project-memory-manager`

## 默认归档位置

- 默认目录：`03_Design/03_Presentation`
- 推荐文件名：`visual-direction.md`
- 归档规则：属于视觉规则与提案文档，不直接进入 `04_Final`
- AI 概念素材默认目录：`03_Design/02_Assets/AI_Concepts/{task-id}`
- AI 概念素材推荐文件名：`ai-concept-brief.md` / `prompts/prompt.md` / `variants/variant-01.png` / `selection-notes.md`
- AI 概念素材归档规则：属于设计资产草案，不直接进入 `04_Final`
- 素材授权登记默认目录：`03_Design/02_Assets`
- 素材授权登记推荐文件名：`asset-register.md`

- 视觉方案必须检查：电商缩略图识别度、搜索/货架远视效果、详情页首图、实物到手识别、印刷工艺适配性、法规信息区预留
- 如果视觉方案依赖高精度工艺（如 4 色以上专色、微缩文字、0.1pt 以下线条），必须标注印刷风险
