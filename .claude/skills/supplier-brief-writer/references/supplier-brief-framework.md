- 适用场景

## 适用场景

供应商 brief 常用于：
- 询价
- 打样
- 可行性确认
- 工艺澄清
- 换供应商重新对齐

不同场景可以共用同一框架，但要在开头写清这次沟通的主目标。

## 必填信息

- 项目背景与产品品类
- 包装结构或盒型
- 关键尺寸
- 目标数量与 MOQ 背景
- 期望材质
- 期望印刷方式
- 期望后道工艺
- 目标交期
- 目标成本或成本压力
- 当前已有文件：刀模、效果图、平面稿、样品照片
- 本次需要供应商回复的事项

## 验收标准

打样或量产相关 brief 必须写清验收重点：

| 阶段 | 供应商需配合确认 |
| --- | --- |
| 白样 | 尺寸、折线、开合、内托、装箱适配、结构强度 |
| 彩样/合同样 | 色差、材质、印刷、烫金/UV/压纹、覆膜、条码可读性 |
| 机上样 | 上机色彩、套印、专色、批量稳定性 |
| 大货首件 | 与签样一致性、批量缺陷、首件放行责任 |
| 大货抽检 | 外观缺陷、数量、批次标识、运输损伤、让步接收条件 |

> **多工序软包装专项**：预制铝箔袋等需经印刷→复合→成型等多工序的产品，各工序公差会逐级累积。打样阶段不能只看成品外观通过，必须要求供应商提供各工序公差值（印刷套准、复合偏移、成型收缩等）并确认累计上限。大货验收时按累计上限逐项核对。（来源：苹果黄芪茶项目——大货验收才发现套印对位偏差，回溯为工序累积公差，RS-006 / DR-007）

## 合规/材料文件需求

根据项目情况要求供应商回复：
- 食品接触材料或内包材说明
- 材料规格、克重、厚度、表面处理
- 工艺外协情况
- 条码印刷可读性风险
- 大货批次和留样方式
- 与企业内部法规/品控复核相关的材料证明

## 供应商最关心的问题

在写 brief 时，优先帮供应商回答这些问题：
- 这是先报价、先打样，还是两者都要
- 当前是概念阶段还是接近定稿阶段
- 哪些要求必须满足，哪些还可讨论
- 成本、交期、效果、环保哪个优先级最高
- 是否已有参考样或参考竞品
- 谁来审批，预计几轮反馈
- 样品通过的标准是什么，大货首件由谁放行

## 常见错误

- 只给效果图，不给关键规格
- 把开放项写成已锁定要求
- 没说明成本和交期优先级
- 不写回复截止时间和需要回复的字段
- 没写验收重点，导致供应商和设计师对“样品成功”理解不同
- 只写“按效果图做”，没有写白样、彩样和大货验收标准
- 涉及内包材或食品接触材料，却没有要求供应商回复材料说明
- 预制铝箔袋等多工序软包装，打样只看成品外观，未要求供应商量化各工序公差累计值（来源：苹果黄芪茶套印对位偏差，RS-006）
