# Decision Log — 决策日志

> 记录设计 / 方案变更的决策理由与触发来源，避免"改了忘了为什么"。
> 对应 skill：`design-version-tracker`

| 日期 | 决策项 | 变更内容 | 触发来源 | 理由 | 影响 | 决策人 |
|------|--------|---------|---------|------|------|--------|
| <YYYY-MM-DD> | <结构 / 工艺 / 视觉 / 材质> | <从 A 改为 B> | [客户 / 成本 / 生产 / 合规 / 评审] | <为什么改> | [成本 / 进度 / 生产] | <人> |

## 填写说明

- **触发来源**：为什么会改（客户反馈、降本、产能、法规、评审意见等）
- **理由**：当时为什么这么定，便于日后追溯
- 关键工艺变更（如烫金 → UV、单盒 → 天地盖）务必记录，防止反复
