# v0.1.1 - Demo Template Expansion

Date: 2026-06-18

Repository: `lihuluu/packging-os`

## Summary

Expanded the demo project into a complete 9-layer blank template set.

## What's Included

`examples/demo-tea-project/` grew from 5 minimal files to 21 blank templates covering the full Layer 0-8 workflow:

- **Layer 0** — `project-memory-card.md`, `project-tracker.md`, `decision-log.md`
- **Layer 1** — `packaging-brief.md`, `brief-decomposition.md`
- **Layer 2** — `positioning-summary.md`, `concept-directions.md`
- **Layer 3** — `structure-decision.md`
- **Layer 4** — `visual-direction.md`, `asset-register.md`
- **Layer 5** — `material-finishing-plan.md`
- **Layer 6** — `prepress-review.md`, `compliance-review.md`, `sample-acceptance-record.md`, `proofing-record.md`
- **Layer 7** — `proposal-outline.md`, `supplier-brief.md`, `quotation-comparison.md`, `packaging-cost-summary.md`
- **Layer 8** — `project-retrospective.md`
- `README.md` with usage guide and placeholder conventions

Each template has structured fields, placeholders (`<必填>` / `[选项]`), cost-evidence levels `[C0]-[C5]`, and compliance/asset/sample status slots. The project header uses 山隐·东方高端茶 as an example placeholder.

## Scope

Examples and documentation only. No changes to core skills, routing, `CLAUDE.md`, or template field definitions. No breaking changes.

## Why

The v0.1.0 demo only covered the first 1-2 layers, so it could not serve as a real reference example. This patch completes the project skeleton so users can see the full 9-layer workflow shape and copy it into real projects.
