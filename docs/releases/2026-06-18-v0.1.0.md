# v0.1.0 - Initial Public Release

Date: 2026-06-18

Repository: `lihuluu/packging-os`

## Summary

Packging OS is now available as a public repository.

This first release turns the local packaging-design studio workflow into a shareable Claude Skills operating system. It includes the 9-layer workflow, reusable skill modules, packaging templates, public knowledge notes, release-safety guardrails, and a fictional demo tea project.

## What's Included

- 18 Claude Skills covering brief decomposition, research, concepts, structure, visual systems, visual direction validation, material and finishing advice, prepress checks, supplier briefs, quotation comparison, project tracking, retrospectives, and knowledge synthesis.
- `CLAUDE.md` as the execution guide for working with Packging OS.
- `README.md` as the human-facing system overview and usage entry.
- Shared standards under `.claude/references/`, including cost evidence, compliance and asset authorization, file naming, sample acceptance, and domain strategy.
- Public reusable knowledge under `Workspace/Knowledge/Library/`.
- Reusable templates under `Workspace/Templates/`.
- A fictional demo project under `examples/demo-tea-project/`.
- Release safety docs under `docs/`, including GitHub release audit, publish next steps, and license notes.
- MIT license.

## Why This Release Matters

This release separates the reusable operating system from private client work.

The public repository now contains the method, routing, templates, and examples needed to understand and adapt Packging OS. Real project records, supplier information, quotes, local settings, production artwork, and operational working state remain outside the public release surface.

## Safety Boundaries

The repository intentionally excludes:

- `.claude/settings.local.json`
- `.obsidian/`
- real project folders under `Workspace/Projects/`
- `Workspace/Knowledge/Operations/`
- AI concept draft folders
- final production artwork
- source design files, PDFs, images, and proofing assets unless explicitly sanitized

## Verification

Before publication, the release surface was checked for:

- external API token patterns
- Bearer token patterns

## Known Limitations

- This is an early public release candidate, not a polished package installer.
- The current default category is tea packaging.
- Category-specific modules for non-tea packaging are not yet specialized.
- The existing internal README is detailed and operational; a shorter GitHub-facing onboarding page may be refined in a later release.
- Local workspace validation still reports private-workspace housekeeping issues that are intentionally outside the public repository surface.
