# AI 概念素材层设计规格

## 背景

Packaging OS 的视觉方向单已经包含“图像/图形”章节，但当前多为描述性判断，尚未把 AI 生图能力纳入项目资产流。用户确认采用“素材草案级别”的能力：AI 负责生成可筛选的概念图像和图形草案，设计师仍负责筛选、重绘、矢量化、印刷适配和最终授权复核。

## 目标

把视觉方向单中的图像/图形部分升级为可执行的 AI 概念素材任务，并让生成结果进入项目资产目录和素材授权登记表。

## 非目标

- 不让 AI 生成物直接进入 `04_Final`。
- 不把 AI 图默认标记为可商用或可量产。
- 不生成或替代 Logo、认证标志、真实检测证明、真实产地照片或法规证据。
- 不在视觉层输出真实成本，只允许标注印刷复杂度或工艺成本压力 [C0]。

## 设计

### 1. 职责边界

| 模块 | 新职责 |
| --- | --- |
| `visual-system-builder` | 在视觉方向单中定义图像/图形策略，并拆成 AI 概念素材任务。 |
| `illustration-generator` | 根据任务生成插画/图形简报、提示词和多个草案变体，输出到项目资产目录。 |
| `asset-register.md` | 登记 AI 草案的生成工具、提示词路径、用途、状态、确认人和限制。 |
| `packaging-os` | 在视觉层路由中识别 AI 概念素材任务，明确它属于视觉层资产流。 |

### 2. 项目输出结构

```text
03_Design/02_Assets/
├── asset-register.md
└── AI_Concepts/
    └── {task-id}/
        ├── ai-concept-brief.md
        ├── prompts/
        │   └── prompt.md
        ├── variants/
        │   ├── variant-01.png
        │   ├── variant-02.png
        │   └── variant-03.png
        └── selection-notes.md
```

### 3. 视觉方向单新增字段

在“图像或图形”之后增加“AI 概念素材任务”：

| 字段 | 含义 |
| --- | --- |
| 任务 ID | 稳定引用，如 `AI-IMG-001`。 |
| 素材类型 | 主视觉、辅助图形、底纹、印章、图标、风格变体。 |
| 生成目的 | 用于提案筛选、方向验证、设计师重绘参考等。 |
| 输入依据 | 来自概念、结构、色彩、字体、渠道或包装面任务。 |
| 输出数量 | 默认 3-5 个变体。 |
| 禁止内容 | 文字、水印、竞品元素、真实产地冒充、证书/标志。 |
| 归档路径 | `03_Design/02_Assets/AI_Concepts/{task-id}/`。 |
| 资产状态 | 默认 `AI 生成草案 / 仅参考 / 不可直接终稿`。 |

### 4. 授权与风险规则

- AI 草案必须记录生成工具、提示词路径、生成日期、后期处理、用途和确认人。
- 未完成企业内部规则或平台授权确认前，授权状态不得写为 `已确认`。
- AI 草案进入设计执行后，应登记“采用为参考”“设计师重绘/加工”或“淘汰”。
- 生产前仍需通过 `prepress-checker` 复核印刷适配、合规信息区和素材授权状态。

### 5. 状态流

```text
待生成 -> 已生成 -> 待筛选 -> 采用为参考 / 淘汰 -> 设计师重绘或加工 -> 生产前授权复核
```

## 验证方式

- 治理校验应通过 `sh .claude/skills/packaging-os-maintainer/scripts/validate-packaging-os.sh`。
- `visual-system-builder`、`illustration-generator`、`packaging-os`、README 和测试样例应表达一致的职责边界。
- 模板中不得出现把 AI 草案直接标记为最终交付的字段。
